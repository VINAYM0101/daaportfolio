Design and Analysis of Algorithms

1. Kinds of Problems We See in Nature

  • Recursion: Break down a problem into simpler instances (e.g., Fibonacci sequence, Tower of Brahma).
  • Iteration: Solve problems step by step using loops (e.g., sum of numbers, traversing arrays).
  • Backtracking: Try all solutions while discarding invalid options (e.g., N-Queens problem, maze navigation, Sudoku).

2. Space and Time Complexity

  • Time Efficiency: Time taken by an algorithm to complete.
  • Space Efficiency: Additional space used by the program.
  • Growth rates (e.g., O(1) for constant time, O(n²) for quadratic time) help analyze efficiency.

3. Takeaway from Different Design Principles

  • Decomposition: Break complex problems into manageable sub-problems to simplify solutions.
  • Pattern Recognition: Identify recurring structures to apply reusable solutions.
  • Abstraction: Focus on the essential details by ignoring irrelevant complexities.
  • Brave and Cautious Travel: Balance exploring risky options with safe alternatives for optimization.
  • Pruning: Eliminate unnecessary possibilities early (e.g., alpha-beta pruning in search trees).
  • Lazy Propagation / Evaluation: Delay computations until absolutely necessary to save resources.
  • Sliding Window: Efficiently process subsets of data with a dynamic range (e.g., in substring problems).
  • Level Order Traversal: Traverse hierarchical structures level by level for breadth-first exploration.
  • Hierarchical Data: Represent data in tree-like structures for better organization and access.
  • Edge Relaxation: Gradually reduce edge weights to find shortest paths (e.g., in Dijkstra’s algorithm).
  • Balancing and Rotations: Maintain balanced structures for efficient operations (e.g., AVL trees).
  • Kleene Closure: Handle repetitive patterns or sequences effectively in automata and regex.
  • Pre-Computing: Compute results in advance to speed up query responses.
  • Parental Dominance: Ensure parent nodes dominate children for ordered structures (e.g., heaps).
  • Prefix and Suffix: Utilize pre-calculated prefix or suffix information to solve range-based problems.
  • Partitioning: Divide data into subsets to simplify and optimize computations (e.g., quicksort partition).
  • Bit Manipulations: Exploit binary operations for compact and efficient data processing.
  • Memoization: Store already computed results to avoid redundant calculations (e.g., Fibonacci series).
  • Invariants: Maintain consistent properties throughout an algorithm’s execution for correctness.
  • Shortest Path Trees: Build trees from source nodes to minimize path costs (e.g., Prim’s, Dijkstra’s).

4. Hierarchical Data and Data Structures

  • Binary Trees: Data stored hierarchically (parent connected to child).
  • Binary Search Trees (BST): Enable efficient searching and insertion.
  • Self-Balancing Trees: Ensure consistent performance (e.g., Heap, AVL Trees, Red-Black Trees).
  • Heaps: Suitable for priority queues.
  • Tries: Useful for autocomplete or dictionary lookups but take more space.

5. Array Query Algorithms

  • Prefix Sum Arrays: Allow quick cumulative sum computations.
  • Fenwick Trees: Provide efficient prefix sums and updates.
  • Segment Trees: Offer range queries and updates.
  • Lookup Tables: Allow fast retrieval of precomputed results.

7. Sorting and Searching Fundamentals

  • Sorting Algorithms:

    • Bubble Sort: Repeatedly compares adjacent elements and swaps them if they are in the wrong order.Use cases: Small datasets, Data that’s almost sorted.
    • Selection Sort: Finds the smallest (or largest) element in the list and places it at the beginning (or end).Use cases: Memory-limited scenarios, Small datasets.
    • Insertion Sort: Builds the sorted list one element at a time by inserting each new element into its correct position.Use cases: Dynamic sorting(e.g., maintaining a sorted list), Nearly sorted data, Small datasets.
    • Merge Sort: Divides the list into smaller sublists until each has one element, then merges them back together in sorted order.Use cases: Large datasets, External sorting(e.g., sorting files on disk), Stable sorting.
    • Quick Sort: Picks a "pivot" element and divides the list into two parts: smaller than the pivot and larger than the pivot.Use cases: General-purpose sorting, In-memory sorting, Performance-critical applications.
    • Heap Sort: Uses a heap (a special tree structure) to repeatedly extract the largest (or smallest) element and rebuild the heap.Use cases: Memory constraints(in-place sorting), Guaranteed time complexity, Priority queues(e.g., job scheduling).

  • Searching Algorithms:

    • Linear Search: Checks each element in the list one by one until it finds the target value or reaches the end.Use cases:When the list is unsorted, When the list is small.
    • Binary Search: Works only on sorted lists.epeatedly divides the list in half, checking if the target value is in the left or right half.Use cases:When the list is sorted, You need a faster search on large datasets.
    • Hash-Based Search: Uses a hash table for near-instant lookups.Use cases: Applied in password validation and database indexing.
    • DFS/BFS: Depth-First Search and Breadth-First Search are graph-based search techniques.Use cases: Used in navigation systems and network troubleshooting.

8. Graph Algorithms for Network Problems

  • Minimum Spanning Trees (MST):

    Kruskal’s and Prim’s algorithms optimize networks like electrical grids.

  • Shortest Path Algorithms:

    • Dijkstra’s Algorithm: Vital for routing and navigation systems.
    • Bellman-Ford Algorithm: Handles graphs with negative weights.

  • BFS:

    Web Crawling, Broadcasting in Networks, Solving Puzzles or Games, Level-wise Exploration.

  • DFS:

    Network Analysis, Finding Connected Components, Pathfinding and Maze Solving, Topological Sorting, Cycle Detection.
